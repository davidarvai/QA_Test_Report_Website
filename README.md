# Manual QA Test Report – Website Bug Findings

## 📌 Overview

This repository contains a manual QA test report for an existing (or in-development) website.

The purpose of this document is to record functional issues, validation problems, and user experience inconsistencies discovered during exploratory manual testing.

This is not a development project, but a QA report based on real user-flow testing.

---

## 🧪 Testing Scope

The following areas were manually tested:

* User registration process
* Login functionality
* Password reset flow
* Profile page behavior
* Checkout / order process
* Input field validations (email, phone number, password strength)
* Search functionality
* UI/UX behavior and navigation

---

## 🐞 Summary of Findings

During testing, multiple issues were identified, including:

* Missing or weak input validation (email, phone number, password strength)
* No confirmation message after registration
* Password recovery process not functioning correctly (no email received)
* Profile page crash after browser refresh
* Lack of character limits in input fields
* UI/UX inconsistencies (missing navigation elements, loading delays)
* Missing or unclear user feedback messages

---

## 🎯 Conclusion

The website shows functional core features; however, several critical and high-severity issues were identified that impact usability, data validation, and system stability.

Key areas requiring improvement:

* Input validation and data integrity
* Authentication and password recovery flows
* System stability (profile page crash issue)
* User feedback and confirmation messages
